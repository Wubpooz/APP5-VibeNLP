Vibe Coding

A new way to build software in 2026

This concept is important because it is how most of you will build your Demo Day projects. Let's understand what it means and where it is heading.

What is Vibe Coding?

"There's a new kind of coding I call 'vibe coding', where you fully give in to the vibes, embrace exponentials, and forget that the code even exists."

— Andrej Karpathy, February 2025

In practice, this means:

  • You describe what you want in natural language
  • AI writes the code for you
  • You run it, see errors, paste them back to AI
  • You iterate until it works — often without reading the code
Andrej Karpathy coined this term in a tweet that went viral. He is the former head of AI at Tesla and a Stanford professor -- when he says something, the industry listens. Note: vibe coding is NOT the same as professional software engineering. It is great for prototypes and personal projects.

The Evolution: Agentic Engineering

Vibe Coding
Feb 2025
→
Agentic Engineering
2026

Vibe Coding

  • Human describes, AI writes
  • No code review
  • Works for prototypes
  • Breaks on complex systems

Agentic Engineering

  • AI agents do implementation
  • Human owns architecture + quality
  • Code review is essential
  • Scales to production systems

Karpathy now says vibe coding is "passé" — the real skill is knowing
what to build and how to verify what the AI built.

This is an important distinction. Vibe coding was a fun idea but it does not scale. Agentic engineering is the mature version: you are the architect, AI is the builder. For the Demo Day project, students should aim for agentic engineering, not vibe coding. That means: define your architecture, write clear specs, let AI implement, then REVIEW. The skill that matters is judgment, not typing speed.

The Simplest Setup

Don't need a full stack? Use the minimal approach:

Gradio App on HF Spaces
Python + Gradio + LLM API
Everything in one place 
import gradio as gr
from openai import OpenAI

client = OpenAI()

def analyze_sentiment(text):
    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[{"role": "user",
                   "content": f"Analyze the sentiment: {text}"}]
    )
    return response.choices[0].message.content

demo = gr.Interface(fn=analyze_sentiment, inputs="text", outputs="text")
demo.launch()

10 lines of Python → deployed web app with a shareable URL

Show this code. It is a complete, deployable NLP application in 10 lines. Obviously the Demo Day project should be more sophisticated, but this demonstrates how low the barrier to entry is. Students have no excuse not to ship something. Gradio handles the entire UI: text input, submit button, output display, loading states.
