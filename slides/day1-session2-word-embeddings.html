Classical Text Representations

The "pre-embedding" era

One-Hot Encoding

Each word = a vector with a single 1, rest all 0s

Vocabulary: [cat, dog, fish, bird, tree]

cat  = [1, 0, 0, 0, 0]
dog  = [0, 1, 0, 0, 0]
fish = [0, 0, 1, 0, 0]
bird = [0, 0, 0, 1, 0]
tree = [0, 0, 0, 0, 1]
  • Simple and unambiguous
  • Problem 1: vector size = vocabulary size (100K+ dimensions!)
  • Problem 2: no notion of similarity — cos(cat, dog) = 0
This is the simplest possible representation. Used in many early ML systems. The key problem: "cat" and "dog" are just as different as "cat" and "tree". Real vocabulary sizes: English ~170K words, so vectors are huge and sparse.

Bag of Words (BoW)

Represent a document by word frequency counts

Doc 1: "the cat sat on the mat"
Doc 2: "the dog sat on the log" 
       cat dog sat on the mat log
Doc 1: [1,  0,  1,  1, 2,  1,  0]
Doc 2: [0,  1,  1,  1, 2,  0,  1]
  • Better: we can now compare documents
  • Lost: word order — "dog bites man" = "man bites dog"
  • Problem: common words ("the", "is") dominate the counts
"Bag" literally means we throw away the order — just count. This is still useful as a baseline. Scikit-learn's CountVectorizer does exactly this. But "the" appearing twice doesn't tell us much about the document.

TF-IDF

Term Frequency × Inverse Document Frequency

TF-IDF(t, d) = TF(t, d) × log( N / DF(t) )

High TF-IDF

Word is frequent in this doc
but rare across all docs

→ discriminative, informative

Low TF-IDF

Word appears everywhere
("the", "is", "and")

→ not useful for distinguishing

Still widely used today: search engines, document retrieval, text classification baselines

The intuition: a word that appears a lot in one document but rarely in others is probably important for understanding what that document is about. "the" gets low weight, "quantum" in a physics paper gets high weight. TF-IDF is still the backbone of many production search systems.

n-grams

Capturing local word order by looking at sequences

Sentence: "I love natural language processing"

Unigrams (n=1)

["I", "love", "natural", "language", "processing"]

Bigrams (n=2)

["I love", "love natural", "natural language", "language processing"]

Trigrams (n=3)

["I love natural", "love natural language", "natural language processing"]

  • Captures: local context, common phrases
  • Problem: vocabulary explodes — Vn possible n-grams
  • Foundation of n-gram language models: P(wordt | wordt-n+1...wordt-1)
n-grams add back some of the word order that Bag-of-Words loses. But the vocabulary grows explosively. If V=100K, bigrams = 10 billion possible pairs. This is a data sparsity problem: most n-grams never appear in your training data.

Method Captures similarity? Word order? Dimensionality

Word Embeddings Yes! Yes (contextual) 50–300 (compact)

We need: dense, low-dimensional, meaning-aware representations

This table summarizes everything we've seen so far. The last row is where we're heading. Word embeddings solve all these problems. This is the breakthrough that changed NLP forever.

The Key Insight

Distributional Semantics

"You shall know a word by the company it keeps."

— J.R. Firth (1957)

The Distributional Hypothesis:

Words that appear in similar contexts
tend to have similar meanings.

"I adopted a ___ from the shelter"

cat? dog? rabbit? → these words must be related!

This is the most important idea in this entire session. Firth's insight from 1957 is the foundation of ALL modern word embeddings. Word2Vec, GloVe, and even BERT are all built on this principle.

Context Windows

A sliding window captures which words appear near each other

Source: Jay Alammar, "The Illustrated Word2Vec" — jalammar.github.io

Window size (typically 2–10) controls how much context we consider

The sliding window moves through the text, one word at a time. For each position, we look at the center word and its neighbors. This is how we build the training data for Word2Vec.

Word2Vec

Mikolov et al., 2013 — Google

Word2Vec: The Breakthrough

  • Published by Tomas Mikolov et al. at Google (2013)
  • Learn dense vector representations (50–300 dimensions)
  • Trained on context prediction — the distributional hypothesis in action
  • Two architectures: Skip-gram and CBOW

Key idea: words are not just symbols — they live in a continuous vector space

Mikolov, T. et al. (2013). "Efficient Estimation of Word Representations in Vector Space." arXiv:1301.3781

Word2Vec is arguably the most influential NLP paper of the 2010s. It made word embeddings practical and accessible. The key insight: you don't need labeled data — just raw text.

Skip-gram

Given a center word, predict the context words

Source: Jay Alammar, "The Illustrated Word2Vec" — jalammar.github.io

Input: "sat" → Output: ["the", "cat", "on", "the"]

Skip-gram is like a fill-in-the-blanks game in reverse: given the answer, predict the surrounding clues. Works better for rare words because each word gets more training signal.

CBOW

Continuous Bag of Words: given context, predict the center word

Source: Jay Alammar, "The Illustrated Word2Vec" — jalammar.github.io

Input: ["the", "cat", "on", "the"] → Output: "sat"

CBOW is the intuitive direction: fill in the blank. "The cat ___ on the mat" — predict "sat". Faster to train, slightly better for frequent words.

Skip-gram vs CBOW

Source: Jay Alammar, "The Illustrated Word2Vec" — jalammar.github.io

Skip-gram CBOW
Direction Center → Context Context → Center
Speed Slower Faster
Rare words Better Worse
Common words Good Slightly better
In practice, Skip-gram with negative sampling (SGNS) is the most widely used. Same architecture, different objective — just swap input and output.

Training Word2Vec

A shallow neural network learns embeddings as a side effect

Source: Jay Alammar, "The Illustrated Word2Vec" — jalammar.github.io

The goal is not the predictions — it's the weight matrix (the embeddings!)

This is a crucial insight: we don't care about the model's predictions. We care about the learned weight matrix — those ARE the word embeddings. After training, throw away the model, keep the weights.

Negative Sampling

The trick that makes Word2Vec practical

Source: Jay Alammar, "The Illustrated Word2Vec" — jalammar.github.io

  • Problem: softmax over 100K+ words is too expensive
  • Solution: binary classification — is this a real pair or a random one?
  • For each positive pair, sample 5–20 negative (random) pairs
Without negative sampling, training would be impractically slow. Instead of asking "which word is it?" (100K-way classification), we ask "is this a real pair?" (binary classification). Much faster.

Training Process

Source: Jay Alammar, "The Illustrated Word2Vec" — jalammar.github.io

1 Look up embeddings for input word and context word
2 Compute dot product → sigmoid → prediction
3 Compare with label (1 = real pair, 0 = negative) → error
4 Update embeddings via backpropagation
Repeat millions of times over the training corpus. Words that frequently appear together get pulled closer in vector space. Words that don't appear together get pushed apart.

What Word2Vec Learns

The magic of the vector space

Words in Vector Space

Source: Jay Alammar, "The Illustrated Word2Vec" — jalammar.github.io

Each dimension captures some latent feature of meaning

Unlike one-hot vectors, each dimension captures something meaningful. The model discovers these features on its own — no human labels needed. Some dimensions might correlate with gender, size, formality, etc.

The Famous Analogy

king − man + woman ≈ queen

Source: Jay Alammar, "The Illustrated Word2Vec" — jalammar.github.io

Vector arithmetic captures semantic relationships!

This is the result that made the world pay attention to word embeddings. The vector from "man" to "king" is approximately the same as from "woman" to "queen". The model learned the concept of "royalty" and "gender" without being told.

Linear Relationships Everywhere

Man ↔ Woman relationships

Comparative ↔ Superlative

Source: Stanford NLP — GloVe: Global Vectors for Word Representation — nlp.stanford.edu/projects/glove

These visualizations from the Stanford GloVe project show consistent patterns. Left: gender relationships form parallel lines. Right: comparative/superlative forms are systematically related. This is not hand-coded — it's entirely learned from text data.

Semantic & Syntactic Patterns

Company ↔ CEO

City ↔ Zip Code

The model learned real-world knowledge just from reading text

Source: Stanford NLP — GloVe: Global Vectors for Word Representation — nlp.stanford.edu/projects/glove

Company-CEO and City-ZIP are remarkable because these are factual relationships. The model doesn't "know" facts — it just learned statistical patterns from text. But those patterns encode real-world knowledge.

Measuring Similarity

Cosine Similarity: angle between two vectors

cos(θ) = (A · B) / (||A|| × ||B||) 
cos(king, queen)   = 0.73  ← high similarity
cos(king, prince)  = 0.71
cos(king, dog)     = 0.15  ← low similarity
cos(cat, dog)      = 0.76  ← both animals!
cos(cat, car)      = 0.12
  • 1.0 = identical direction (same meaning)
  • 0.0 = orthogonal (unrelated)
  • -1.0 = opposite direction
Cosine similarity is the standard metric for comparing word embeddings. We use cosine instead of Euclidean distance because it's length-invariant. These numbers are illustrative — actual values depend on the model and corpus.

Think About It

What other analogies would you expect Word2Vec to capture?

  • Paris − France + Japan = ?
  • Apple − fruit + Samsung = ?
  • doctor − man + woman = ?

That last one is interesting... what does it tell us about bias in training data?

Paris - France + Japan = Tokyo. Apple is tricky — it's ambiguous. doctor - man + woman often = nurse in early models — this reveals gender bias! Good segue: embeddings learn biases present in the training data. This is an active research area: debiasing word embeddings.

GloVe

Pennington, Socher & Manning, 2014 — Stanford

GloVe: Global Vectors

Global Vectors for Word Representation

  • Key difference from Word2Vec:
    uses global co-occurrence statistics, not just local windows
  • Step 1: build a word-word co-occurrence matrix from the entire corpus
  • Step 2: learn vectors such that their dot product approximates the log co-occurrence
wi · wj ≈ log P(i, j)

Pennington, J., Socher, R. & Manning, C.D. (2014). "GloVe: Global Vectors for Word Representation." EMNLP. — nlp.stanford.edu/projects/glove

GloVe combines the best of two worlds: - Global matrix factorization (like LSA) — captures global statistics - Local context window (like Word2Vec) — captures local patterns The name literally means "Global Vectors".

The Key Insight: Co-occurrence Ratios

What really encodes meaning? Not raw counts, but ratios!

Probe word k P(k|ice) P(k|steam) P(k|ice) / P(k|steam)
solid 1.9 × 10⁻⁴ 2.2 × 10⁻⁵ 8.9
gas 6.6 × 10⁻⁵ 7.8 × 10⁻⁴ 0.085
water 3.0 × 10⁻³ 2.2 × 10⁻³ 1.36
fashion 1.7 × 10⁻⁵ 1.8 × 10⁻⁵ 0.96
  • Large ratio: "solid" strongly associated with "ice" vs "steam"
  • Small ratio: "gas" strongly associated with "steam" vs "ice"
  • Ratio ≈ 1: "water" and "fashion" neutral (or irrelevant)

Table from the original GloVe paper (Pennington et al., 2014)

This table is the heart of the GloVe paper. The ratios encode semantic relationships in a way raw counts don't. GloVe's objective: learn vectors whose dot products capture these ratios.

Why Dot Products = Log Counts?

The derivation follows from a simple requirement

1 Goal: find F such that F(wi, wj, w̃k) = Pik / Pjk
2 Ratios come from differences: F((wi - wj) · w̃k) = ratio
3 What converts differences → ratios? The exponential!
exp(a - b) = exp(a) / exp(b)
4 Take log of both sides → wi · w̃k = log(Xik) + biases

The objective function isn't arbitrary — it's mathematically inevitable!

This derivation is what makes GloVe elegant. Each step is forced by mathematical constraints. The exponential is the ONLY function that converts addition to multiplication.

Co-occurrence Matrix

Count how often words appear near each other in the corpus

         cat   dog   sat   the   mat
cat    [  0,    2,    5,   10,    3 ]
dog    [  2,    0,    4,    8,    1 ]
sat    [  5,    4,    0,   12,    6 ]
the    [ 10,    8,   12,    0,    7 ]
mat    [  3,    1,    6,    7,    0 ]
  • "cat" and "sat" co-occur 5 times → they're related
  • This matrix encodes the entire corpus in one structure
  • GloVe factorizes this matrix into low-dimensional vectors
Numbers are illustrative, not real. The matrix is symmetric: co-occurrence(cat, sat) = co-occurrence(sat, cat). For a real corpus, this matrix would be huge (|V| x |V|) and very sparse. GloVe learns a compact representation that preserves the important information.

The GloVe Objective Function

Minimize the weighted squared error of dot products vs log counts

J = Σi,j f(Xij) (wi · w̃j + bi + b̃j - log Xij)²

Key Components

  • wi · w̃j: dot product of word vectors
  • bi, b̃j: bias terms
  • log Xij: target (log co-occurrence)

Weighting Function f(x)

  • f(0) = 0: ignore zero counts
  • Non-decreasing: more co-occurrences → more weight
  • Capped: don't over-weight "the", "a"

f(x) = (x/xmax)α if x < xmax, else 1   (typically α = 0.75, xmax = 100)

The weighting function f is crucial. Without it, common words like "the" would dominate the objective. The 0.75 exponent was found empirically to work best.

The Deep Connection

Word2Vec and GloVe are more similar than they appear

Levy & Goldberg (2014) proved that Word2Vec Skip-gram with Negative Sampling is implicitly factorizing a shifted PMI matrix!

wi · w̃j = PMI(i, j) - log k

where k = number of negative samples

  • Word2Vec: learns this factorization implicitly via SGD
  • GloVe: learns this factorization explicitly via matrix objective
  • Both capture the same underlying semantic structure!

Levy, O. & Goldberg, Y. (2014). "Neural Word Embedding as Implicit Matrix Factorization." NeurIPS.

This is a landmark paper in understanding word embeddings. It unified the "prediction-based" and "count-based" views. PMI = Pointwise Mutual Information, a classic measure from information theory.

Word2Vec vs GloVe

Word2Vec GloVe
Method Predictive (neural network) Count-based (matrix factorization)
Uses Local context windows Global co-occurrence statistics
Training Online (iterates through text) Offline (pre-computes matrix)
Performance Similar Similar
Pre-trained Google News (3M words) Wikipedia + Gigaword (6B tokens)

In practice, both produce high-quality embeddings — GloVe is often slightly faster to train

The debate "which is better?" has largely been settled: they're comparable. GloVe tends to be preferred in academic research partly because of Stanford's influence. Word2Vec with negative sampling is very popular in industry. Both are available as pre-trained models you can download and use immediately.

Properties & Limitations

What static embeddings can and cannot do

Window Size Matters

Source: Jay Alammar, "The Illustrated Word2Vec" — jalammar.github.io

Small window (2–3)

Captures syntactic similarity
"big" ↔ "large" ↔ "huge"

Large window (8–10)

Captures semantic/topical similarity
"dog" ↔ "puppy" ↔ "veterinarian"

This is a practical consideration when training embeddings. Small windows: words that can replace each other (synonyms, same POS). Large windows: words that appear in the same topic or domain.

Bias in Embeddings

Embeddings learn the biases present in the training data

man → computer programmer
woman → homemaker

doctor − man + woman ≈ nurse
  • These reflect historical stereotypes in training text
  • Active research area: debiasing word embeddings (Bolukbasi et al., 2016)
  • Important lesson: data is not neutral
This is a crucial topic for responsible AI. The model isn't "sexist" — it's reflecting patterns in the text it was trained on. But when deployed in hiring systems, recommendations, etc., these biases cause real harm. Bolukbasi et al. paper: "Man is to Computer Programmer as Woman is to Homemaker?"

The Polysemy Problem

Static embeddings: one vector per word, regardless of context

"I went to the bank to deposit money"

"I sat on the bank of the river"

Word2Vec / GloVe: same vector for both!

Solution → Contextual embeddings (ELMo, BERT, GPT)

Different vector for each usage of the word — Day 2 & 3 material

This is THE fundamental limitation of Word2Vec and GloVe. Static = one vector forever. But words have multiple meanings. The solution came in 2018 with ELMo and BERT: contextual embeddings where the vector depends on the surrounding context. This is the bridge to Transformers — covered on Day 2.

Using Embeddings in Practice

From theory to code

Pre-trained Embeddings

You don't need to train from scratch — download and use

Model Corpus Vocabulary Dimensions
Word2Vec Google News 3M words 300
GloVe Wikipedia + Gigaword 400K words 50, 100, 200, 300
GloVe (large) Common Crawl 2.2M words 300
FastText Wikipedia 1M words 300
In practice, you almost always start with pre-trained embeddings. Training from scratch requires massive corpora and compute. GloVe 6B (Wikipedia) is a great starting point for experiments. FastText is worth mentioning — it handles subwords, useful for rare words.

Using Word2Vec in Python

import gensim.downloader as api

# Load pre-trained Word2Vec (Google News, 300d)
model = api.load("word2vec-google-news-300")

# Find similar words
model.most_similar("python")
# [('pythons', 0.71), ('snake', 0.65), ('lizard', 0.58), ...]

# Word analogy: king - man + woman = ?
model.most_similar(positive=["king", "woman"],
                   negative=["man"], topn=1)
# [('queen', 0.71)]

# Cosine similarity
model.similarity("cat", "dog")     # 0.76
model.similarity("cat", "car")     # 0.12

Library: gensim — pip install gensim

Live demo opportunity: run this code in front of students. Let them suggest words and analogies to try. gensim is the standard library for Word2Vec in Python.

Where Embeddings Are Used

  • Text Classification — embeddings as input features
  • Search & Information Retrieval — semantic search beyond keywords
  • Recommendation Systems — Spotify, Airbnb use embedding-based recommendations
  • Transfer Learning — pre-trained embeddings boost downstream tasks
  • Foundation for Transformers — BERT, GPT start with embeddings
Embeddings are not just an academic exercise — they're everywhere in production. Spotify's "Word2Vec for music" treats songs as words and playlists as sentences. Airbnb uses listing embeddings for search ranking. And of course, every LLM starts by embedding input tokens.
