# Présentations de partage de connaissances en groupe

## Informations générales

- **Dates** : 9 mars - 20 mars 2026 (4 séances)
- **Format** : Présentations en groupe, 20 minutes par groupe
- **Sujets** : 4, de difficulté croissante

## Description des sujets

Les éléments suivants sont des directions de référence, pas une liste à couvrir exhaustivement. Chaque groupe choisit ses points clés selon sa propre compréhension — la profondeur compte plus que l'exhaustivité.

### Sujet #1 : Boîte à outils startup (09/03)

Préparer le projet NLP pour le Demo Day, partager des outils pratiques et des retours d'expérience :

- Gestion de projet (ex. Linear, GitHub Projects)
- Obtention de tokens API LLM gratuits
- Outils de Vibe Coding
- Déploiement frontend (ex. Vercel)
- Hébergement backend

### Sujet #2 : Claude Code en profondeur (13/03)

Maîtriser les fonctionnalités avancées et les workflows de Claude Code :

- MCP (Model Context Protocol)
- Création et utilisation de Skills
- Ingénierie de prompts et CLAUDE.md
- Techniques de Tool Use
- Environnements de terminal (ex. Conductor, tmux)

### Sujet #3 : Attention Is All You Need (19/03)

Comprendre le cœur de l'architecture Transformer :

- Mécanisme de Self-Attention (Query, Key, Value)
- Multi-Head Attention
- Positional Encoding
- Architecture Encoder-Decoder
- Du Transformer à BERT, GPT

### Sujet #4 : OpenClaw et écosystème Agent (20/03)

Comprendre la conception et les tendances de développement des AI Agents :

- Configuration et utilisation d'OpenClaw
- Philosophie de conception et architecture d'OpenClaw
- Écosystème de plateformes (ex. Moltbook)
- Protocoles et tendances Agent-to-Agent (A2A)

## Critères de notation

Ceci n'est pas un exposé académique traditionnel — c'est un exercice de **partage de connaissances et de réflexion sur l'apprentissage**.

| Dimension | Description | Pondération |
|-----------|-------------|-------------|
| Transmission des connaissances | Le public a-t-il réellement appris quelque chose ? | 40% |
| Processus et méthodes d'apprentissage | Le groupe a-t-il montré comment il a appris le sujet ? (outils IA utilisés, discussions entre pairs, difficultés rencontrées) | 40% |
| Qualité de la présentation | Expression claire, structure cohérente, respect des 20 minutes | 20% |
